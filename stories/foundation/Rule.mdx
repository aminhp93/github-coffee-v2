# Rule

```tsx
// naming convention

- When naming variables, files, folders, ... name should be meaningful

- Folder name use kebab-case, example :
    ✅ Ok: tag-explorer
    ❌ Not Ok: tagExplorer, TagExplorer, tag_explorer

- Component file use PascalCase, example :
    ✅ Ok: TagExplorer
    ❌ Not Ok: tagExplorer, tag-explorer, tag_explorer

- Variables and functions name use camelCase, example :
    ✅ Ok: tagExplorer
    ❌ Not Ok: TagExplorer, tag-explorer, tag_explorer

- Constants and enum use SNAKE_CASE, example :
    ✅ Ok: TAG_EXPLORER
    ❌ Not Ok: TagExplorer, tag-explorer, tag_explorer
```

```tsx
// folder structure

- Write all service files in the root folder


```

```tsx
// interface rules

- When using api responses in component level, only use necessary data value, omit unused value, example:

    path : webpmp-v5/service/type.ts
    interface UserResponse {
        id:string,
        username:string,
        email:string,
        token:string,
    }

    path : webpmp-v5/modules/component/type.ts
    ✅ Ok
    type User = Omit<UserResponse, "token">

    ❌ Not Ok
    type User = UserResponse

    ❌ Not Ok
    type User =  {
        id:string,
        username:string,
        email:string,
        token:string,
    }



```
